ConfigureRequest	- запрос конфигурации окна пытается заново сконфигурировать размер окна, положение, границы и порядок наложения.Х-сервер генерирует это событие, когда другой клиент инициирует запрос конфигурации окна на окно, вызвав XConfigureWindow (), XLowerWindow (), XRaiseWindow (), XMapRaised (), XMoveResizeWindow (), XMoveWindow (), XResizeWindow (), XRestackWindows (), или XSetWindowBorderWidth ().
MapRequest		- Х-сервер может сообщить MapRequest событии клиентам желающим информацию о желании различных клиентских отображениях окон. Окно считается отображенным, когда запрос окна на обображение будет завершен.Х-сервер генерирует это событие, когда другой клиент инициирует запрос отображения окна на неотображенное окно, override_redirect члена имеет значение False. Клиенты инициировать запросы отображения вызывая XMapWindow (), XMapRaised (), или XMapSubwindows ().
ResizeRequest		- Х-сервер генерирует это событие, когда другой клиент пытается изменить размер указанного окна вызвав XConfigureWindow (), XResizeWindow (), или XMoveResizeWindow ().
CirculateNotify		-
ConfigureNotify	- Х-сервер может сообщить ConfigureNotify событие клиентам желающим информацию о фактическом изменении состояния окна, такие как размер, положение, границы и порядок наложения.Х-сервер создает этот тип событий, когда одно из следующих конфигурации окна на самом деле завершается:
Размер окна, положение, границы, и / или порядка размещения измененяется по вызову XConfigureWindow ().
Положение окна в порядке расположения меняется по вызову XLowerWindow (), XRaiseWindow (), или XRestackWindows ().
Окно перемещается по вызову XMoveWindow ().
Размер окна изменен по вызову XResizeWindow ().
Размер окна и расположение изменено по вызову XMoveResizeWindow ().
Окна отображается и его положение в стеке меняется, вызывая XMapRaised ().
Ширина границы окна меняется, вызывая XSetWindowBorderWidth ().
CreateNotify		- Х-сервер может сообщить CreateNotify событие клиентам желающим информацию о создании окна.Х-сервер генерирует это событие каждый раз, когда клиентское приложение создает окно по вызову XCreateWindow () или XCreateSimpleWindow ().
DestroyNotify		- Х-сервер может сообщить DestroyNotify событие клиентам желающим информацию о уничтожении окна.Х-сервер генерирует это событие каждый раз, когда клиентское приложение уничтожает окно по вызову XDestroyWindow() или XDestroySubwindows().
GravityNotify		- Х-сервер может сообщить GravityNotify событий клиентам желающим информацию о том, когда окно перемещается в связи с изменением размера его родителей.Х-сервер генерирует это событие каждый раз, когда клиентское приложение на самом деле движется дочернее окно в результате изменения размера родительского по вызову XConfigureWindow (), XMoveResizeWindow (), или XResizeWindow ().
MapNotify		- Х-сервер может сообщить MapNotify событий клиентам желающим информацию о том, какие окна отображаются.Х-сервер создает этот тип события, когда клиентское приложение изменяет состояние окна - отображается по вызову XMapWindow (), XMapRaised (), XMapSubwindows (), XReparentWindow (), или в результате save-set обработки.
MappingNotify		- Х-сервер сообщает MappingNotify событие всем клиентам. Не существует механизма для отключения интереса к этому событию.Х-сервер создает этот тип события, когда клиентское приложение успешно вызывает:
XSetModifierMapping (), чтобы указать, какие коды клавиш будут использованы в качестве модификаторов
XChangeKeyboardMapping () для изменения раскладки клавиатуры
XSetPointerMapping () для установки указателя отображения
ReparentNotify		- Х-сервер может сообщить ReparentNotify событий клиентам желающим информацию об изменении родителя окна.Х-сервер генерирует это событие каждый раз, когда клиентское приложение вызывает XReparentWindow () и окно на самом деле изменяет родителя.
UnmapNotify		- Х-сервер может сообщить UnmapNotify событий клиентам желающим информацию о том, какие окна не отображаются.Х-сервер создает этот тип события, когда клиентское приложение переходит в состояние не отображения.
VisibilityNotify		- Х-сервер может сообщить VisibilityNotify событий клиентам желающим любые изменения в видимости указанного окна.Области окна отображаются, если кто-то, глядя на экран может увидеть его.Х-сервер генерирует это событие, когда видимость изменяется. Тем не менее, это событие не генерируется для окон, чей класс InputOnly.
ColormapNotify	- Х-сервер может сообщить ColormapNotify событий клиентам желающим информацию о том, когда палитру изменена и когда палитра установлена ​​или удалена.Х-сервер создает этот тип события, когда клиентское приложение:
Изменяет член структуры палитры XSetWindowAttributes по вызову XChangeWindowAttributes (), XFreeColormap (), или XSetWindowColormap ()
Устанавливает или удаляет палитру по XInstallColormap телефону () или XUninstallColormap ()
ClientMessage		- Х-сервер генерирует ClientMessage события только тогда, когда клиент вызывает функцию XSendEvent ().
PropertyNotify		- Х-сервер может сообщить PropertyNotify событие клиентам желающим информацию об изменениях свойств для указанного окна.
SelectionClear		- Х-сервер сообщает SelectionClear событие клиенту потерявшему права собственности на выбор.Х-сервер создает этот тип события, когда другой клиент утверждает, право собственности на выбор по вызову XSetSelectionOwner ().
SelectionNotify		- Это событие генерируется сервером X в ответ на запрос протокола ConvertSelection при отсутствии владельца для Selection. Когда есть собственник, он должен быть создан владельцем Selection с помощью XSendEvent (). Владелец Selection должен отправить это событие запроса, когда Selection был преобразован и хранится как свойство или при Selection преобразование не может быть выполнена (о чем свидетельствует установление собственности членов нет).
SelectionRequest	- Х-сервер сообщает SelectionRequest событие владельцу Selection.Х-сервер генерирует это событие каждый раз, когда клиент просит Selection преобразования по вызову XConvertSelection () для принадлежащего Selection.
